An alternate method for defining a finite-state machine is with a transition diagram. A transition diagram is a directed graph that contains a node for each state and edges that indicate the transition and output functions. An edge \(\left(s_i,s_j\right)\) that is labeled \(x/z\) indicates that in state \(s_i\) the input \(x\) results in an output of \(z\) and the next state is \(s_j\text{.}\) That is, \(w\left(x, s_i\right)=z\) and \(t\left(x, s_i\right)=s_j\text{.}\) The transition diagram for the parity checker appears in FigureÂ 14.3.4. In later examples, we will see that if there are different inputs, \(x_i\) and \(x_j\text{,}\) while in the same state resulting in the same transitions and outputs, we label a single edge \(x_i, \left.x_j\right/z\) instead of drawing two edges with labels \(\left.x_i\right/z\) and \(\left.x_j\right/z\text{.}\)
